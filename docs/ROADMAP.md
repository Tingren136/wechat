# Roadmap

## 固定前提

- 顶层目标是“半自动化”，不是“一键发文”
- 保留 4 套固定主题，用户手动选择
- 子 skill 必须完整遵循原始流程
- 正文图片默认 `4:3`
- 正文连续纯文字不应超过约 `300` 中文字
- 文章资产必须按“单文章单目录”落盘
- 正文工作流与封面工作流分开推进

## Phase 1: 项目说明固化

- 补齐 README
- 写清楚现状、目标、输入输出
- 写清楚固定规则、确认点、目录约定
- 让其他 AI 可以无上下文接手

## Phase 2: 预览原型产品化

- 清理预览脚本
- 减少对绝对路径的依赖
- 固化文章输入与输出目录
- 保证 4 套预览 HTML 稳定生成
- 保证 4 套发布态 HTML 稳定生成
- 保证预览与发布态产物都落到文章同名目录

## Phase 3: 工作流拆段

- 正文润色阶段
- Markdown 结构化阶段
- 配图规划与提示词打包阶段
- 正文图片生成阶段
- 图片插回正文与阅读节奏检查阶段
- 4 套 HTML 输出阶段
- 发布态导出与草稿箱前检查阶段

## Phase 4: Skill 封装

- 统一 skill 输入参数
- 明确每个确认点
- 输出中间产物与最终产物
- 固化单文章目录结构与命名规则
- 固化“300 字内视觉中断”检查规则
- 固化“短图注 + 圆角显示层裁切”规则
- 准备示例与文档

## Phase 5: GitHub 发布

- 整理项目目录
- 加入示例素材与截图
- 补齐安装与使用说明
- 补齐从草稿到 4 套 HTML 的最小闭环示例

## Deferred

- 封面工作流单独封装
- 封面比例与固定风格库后续再补
